### 🏦 Functional Mocks

| Repository | Description |
|---|---|
| [**fintech-data-mock**](https://github.com/edsonirvana/fintech-data-mock) | Full data pipeline for a fictional fintech (**PayBridge Financeira**) on GCP. PostgreSQL legacy ingestion via Python ETL → Cloud Storage → BigQuery, **dbt** transformations with Medallion Architecture (staging → intermediate → marts), semantic layer in **LookML/Looker**, and governance with RLS, PII masking, and financial reconciliation tests. KPIs: TPV, MDR, NPL, credit exposure, chargeback rate, and net revenue. |
| [**ml-nps-analysis**](https://github.com/edsonirvana/ml-nps-analysis) | Automated NPS comment analysis pipeline using LLM. LangGraph StateGraph with 7 nodes, dynamic model routing (Claude, LLaMA, Mistral, Mixtral), per-category sentiment classification, and per-node automatic retry. **LangGraph · Snowflake Cortex · Python** |
| [**ml-rfm-segmentation**](https://github.com/edsonirvana/ml-rfm-segmentation) | Per-venue RFM user segmentation with quintile scoring — two variants: physical venues and ticketing. Scores computed within each venue (not globally), batch processing of 100 venues at a time. **Pandas · NumPy · Snowflake Feature Store** |
| [**ml-product-classification**](https://github.com/edsonirvana/ml-product-classification) | 3-stage NLP pipeline to classify free-text product names into beverage categories. OneClassSVM for detection, CatBoost for type and category, fuzzy brand matching via rapidfuzz. **CatBoost · TF-IDF · NLTK · rapidfuzz · MLflow** |
| [**ml-user-return-propensity**](https://github.com/edsonirvana/ml-user-return-propensity) | Return propensity prediction per user-venue pair. Per-pair RFM features + global user profile, 3-month batch loading, threshold at 0.70 for precision-focused scoring. **CatBoost · MLflow · scikit-learn** |
| [**ml-churn-classifier**](https://github.com/edsonirvana/ml-churn-classifier) | Binary churn classification pipeline for transactional businesses — built and validated at production scale. 80+ behavioral and financial features engineered from transaction history, with no dependency on CRM or demographic data. **CatBoost · SMOTE · MLflow · scikit-learn** |
